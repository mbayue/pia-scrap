## 2024-07-04 - [Use lxml for BeautifulSoup parsing]
**Learning:** `BeautifulSoup` using the default `html.parser` is noticeably slower compared to the `lxml` parser, particularly when running repeatedly (e.g., parsing thousands of chapters downloaded sequentially or in parallel). However, `lxml` behaves differently when parsing HTML fragments, automatically wrapping them in `<html><body>...</body></html>`. This auto-wrapping behavior can cause nested structures or missing meta tags if the surrounding codebase expects to inject the wrapper manually.
**Action:** When switching to the `lxml` parser in `BeautifulSoup` to improve performance on repetitive HTML parsing, ensure that you handle its auto-wrapping behavior correctly to preserve the exact HTML document structure (e.g. by checking for and injecting missing `<head>` and `<meta>` tags if the auto-wrapper skipped them) and explicitly list `lxml` in `requirements.txt`.
